# Wochit APIs

Enhance your integration by utilizing API calls to efficiently manage drafts and videos, while also enabling the extraction of template category IDs.

# Drafts & Videos

You can enable your users to clone or delete videos and drafts using the following API calls:

# Clone Draft

curl --request POST 'https://api.wochit.com/v1/users/drafts/{id}/clone' \
--header 'Authorization: Bearer <user token>' \
--header 'client-id: <client id>' \
--header 'Content-Type: application/json' \
}

# Delete Draft

curl --request DELETE 'https://api.wochit.com/v1/users/drafts/{id}' \
--header 'Authorization: Bearer <user token>' \
--header 'client-id: <client id>' \
--header 'Content-Type: application/json' \
}

# Clone Video

curl --request POST 'https://api.wochit.com/v1/users/videos/{id}/clone' \
--header 'Authorization: Bearer <user token>' \
--header 'client-id: <client id>' \
--header 'Content-Type: application/json' \
}

# Delete Video

curl --request DELETE 'https://api.wochit.com/v1/users/videos/{id}' \
--header 'Authorization: Bearer <user token>' \
--header 'client-id: <client id>' \
--header 'Content-Type: application/json' \
}

Please note that to access these APIs, you need your Client ID and a user token. You can find the Client ID in your Developer Tools (opens new window). The user token is a unique key generated each time a user accesses the Wochit Video Editor.

  • ID - mandatory. This is a unique identifier of the video or draft.
  • A draft\video can be deleted or cloned only by the user who created it. This is a default setup to help you secure your user's actions. If you need to remove this limitation please contact us (opens new window).

# Response

The response for cloning a draft or a video will include the following JSON structure:

{
  "wochitVideoId": "63ce6eaa7304484941234524",
  "title": "My Video",
  "thumbnail": "https://example.com/direct-link-to-thumbnail"
}

For successful deletion, the response will be a 204 No Content status code.

# Attributes

The API responses include the following attributes:

wochitVideoId string
Wochit unique identifier for this video. You can use it to re-edit the video.

title string
The title of the video. The title is given by the end-user.

thumbnail string
Direct link to the thumbnail of the video. Links have an expiration time of 24 hours so make sure you download the video to your system and use your own storage / CDN.

# Template Categories

Each template in the template gallery belongs to one or more categories. You can choose the entire set of templates you’d like to include in your Video Editor in the admin panel (opens new window) under Templates -> Wochit Template Packages.
Have your own set of templates and categories in the admin panel (opens new window) under Templates -> My Templates -> Create.

Use the template IDs to filter templates when you open the video creator.
To extract the IDs of your template categories using the API, use the following request:

curl --request POST 'https://api.wochit.com/v1/accounts/templates/categories' \
--header 'secret-key: <secret key>' \
--header 'client-id: <client id>' \
--header 'Content-Type: application/json' \
}'

Please refer to the Developer Tools (opens new window) to obtain your Client ID and Secret Key.

The response will be structured in a JSON format and will include the registered and owned categories:

{
  "registered": [
    {
      "id": "62543d82f22d3777bc28986c",
      "name": "Education"
    }
  ],
  "owned": [
    {
      "id": "647f33677387131d481f2d30",
      "name": "My Category"
    }
  ]
}

The response is structured in a JSON format with the following parameters:

registered object[] List of categories selected in the Wochit Admin.

owned object[] Your custom categories created using the Wochit Studio application.

  • id string A Unique identifier of each category. Use the categoryIds property to filter the template gallery.
  • name string The Name of the category. Use the categoryNames property to filter the template gallery.
